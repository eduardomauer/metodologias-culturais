# Privacidade, RGPD e minimização

## Regra principal

Este repositório só deve conter informação pública, minimizada e compatível com proteção de dados.

## Não publicar

Não devem ser publicados:

- contactos pessoais;
- documentos de identificação;
- moradas;
- dados bancários;
- assinaturas;
- dados de saúde;
- dados de menores;
- testemunhos identificáveis sem base jurídica;
- fotografias de pessoas sem autorização adequada;
- georreferenciação sensível associada a pessoas ou comunidades vulneráveis;
- consentimentos ou provas internas completas.

## Minimização

Sempre que possível, usar dados agregados, anonimizados, pseudonimizados ou descrições públicas sem identificação individual.

## Revisão

Qualquer conteúdo com risco de identificação pessoal deve ser removido da camada pública até revisão formal.

## Incidentes

Se for encontrado dado pessoal indevido, o conteúdo deve ser corrigido por via controlada, preservando evidência interna, hash, data, responsável e motivo da correção.
